Resolved Issues

Fixing Prisma 7 Build Errors

The initial Netlify deploy failed during Next.js compilation because the generated Prisma client files were not present in the CI workspace. We made the following changes to ensure a successful build:

  • Build Automation: Modified package.json's build script from next build to prisma generate && next build. This guarantees the client is generated automatically in any clean CI environment before compilation.
  • Prisma 7 Compatibility: Updated prisma/schema.prisma provider to postgresql. In Prisma 7, connection URLs are no longer supported inside the schema file, so we removed the url = env("DATABASE_URL") property, delegating connection settings entirely to prisma.config.ts.
  • PostgreSQL Adapter Migration: Swapped the local SQLite driver adapter in src/lib/prisma.ts for the standard PostgreSQL Pool and WASM-based @prisma/adapter-pg connection wrapper.

The repository changes compile successfully locally using a mock PostgreSQL connection string and have been committed and pushed to the main branch on GitHub.
