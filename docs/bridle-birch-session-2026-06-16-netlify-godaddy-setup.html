Deployment Strategy

Netlify + Supabase + GoDaddy Guide

Because Netlify is serverless, the local SQLite database dev.db cannot be used in production. We created a detailed guide at docs/netlify-godaddy-deployment.md detailing:

  1. Setting up a hosted Supabase (PostgreSQL) transaction-mode database connection.
  2. Modifying the Prisma database provider to use postgresql.
  3. Configuring environment variables (DATABASE_URL and JWT_SECRET) on Netlify.
  4. Pointing the GoDaddy Nameservers to Netlify DNS to automatically handle HTTPS certificates and DNS routing.
Database Parity Recommendation

To avoid swapping the Prisma provider back and forth between SQLite locally and PostgreSQL in production, we recommend connecting the local development environment directly to a Supabase project or setting up a local Postgres container.
