Core Research Findings

Engraving Overlay Render Solutions

To move away from flat text layers on top of wood and glass mockups, we researched four core personalization architectures:

  • Client-Side SVG Displacement (Recommended for MVP): Uses SVG filters (<feDisplacementMap>) to dynamically warp custom text boundaries along the texture coordinates of a base image, blended using CSS modes (like mix-blend-mode: multiply). Costs nothing and loads instantly.
  • Client-Side WebGL (Three.js): Renders product assets as true 3D models with normal/bump mapping to catch lighting dynamically. High implementation cost, but supports 3D cylindrical wrap.
  • SaaS Mockup Rendering (Dynamic Mockups API): Uses uploaded Photoshop templates with Smart Objects, warp maps, and layer styles. Replacing smart object data dynamically generates fully photorealistic mockups on the fly. Requires subscription and introduces 1.5s latency.
  • Server-Side Image Processing (Sharp/ImageMagick): Executes rendering on edge/server instances to warp and cache product preview images. Complex infrastructure dependency.

A comprehensive analysis was created and saved to docs/engraving-preview-research.md.

02

Proposed Implementation Plan

Client-Side SVG Displacement Plan

We drafted a project implementation plan details-first at implementation_plan.md. The core proposal consists of:

  1. Generating Local Heightmaps: Storing high-contrast, blurred grayscale images representing the wood/glass texture mapping layers in public/images/heightmaps/.
  2. Modifying Preview Component: Rewriting src/components/engraving-preview.tsx to overlay a dynamic SVG text layer referencing the displacement filter map, applying multiply blending for wood burn and overlay/soft-light blending for frosted glass etches.
Zero Cost & Instant Loading

By executing displacement entirely in the client's browser, the storefront maintains absolute speed and zero server operational costs, providing immediate customer feedback without visual jitter.
